## 1. О чём этот проект

**Предметная область:** магазин обуви ООО «Обувь».

**Технологии:**
- **PostgreSQL** — база данных (работа через **pgAdmin 4**)
- **Python** — язык приложения
- **tkinter** — окна, кнопки, поля ввода
- **psycopg2** — связь Python ↔ PostgreSQL
- **Pillow (PIL)** — загрузка и изменение картинок

**Роли пользователей:**
| Роль | Что может |
|------|-----------|
| Гость | Только смотреть товары (без поиска) |
| Авторизированный клиент | Только смотреть товары |
| Менеджер | Товары + поиск/сортировка/фильтр + просмотр заказов |
| Администратор | Всё выше + добавление/изменение/удаление товаров и заказов |

## 3. Файлы в корне проекта

### config.py

**Назначение:** настройки подключения к PostgreSQL.

```python
DB_CONFIG = {
    "host": "localhost",
    "port": 5432,
    "dbname": "shoe_store_demo",   # имя твоей базы
    "user": "postgres",
    "password": "postgres",        # СВОЙ пароль от PostgreSQL
}
```

**Когда менять:** перед каждым запуском `app.py`, если другой пароль или имя БД.

---

### app.py

**Назначение:** всё приложение в одном файле (модули 2 + 3 + 4).

**Запуск:**
```bash
pip install psycopg2-binary Pillow
python app.py
```

**Порядок работы программы:**
1. `main()` — создаёт окно, ставит иконку, шрифт
2. `show_login()` — экран входа (первый)
3. После входа → `show_products()` — список товаров
4. Кнопка «Заказы» → `show_orders()` (модуль 4)
5. «Выход» → снова `show_login()`

### requirements.txt (в модуль 2)

```
psycopg2-binary

```

**Когда использовать:** один раз перед экзаменом: `pip install -r requirements.txt`

## 4. Модуль 1 — База данных

### Цель модуля

- Создать БД по предметной области
- 3 нормальная форма, первичные и внешние ключи
- Импорт данных из файлов заказчика
- ER-диаграмма в PDF

### Что сдаёшь

1. База `shoe_store_demo` с данными
2. SQL-скрипт (или `backup_bd.sql`)
3. ER-диаграмма PDF

### Таблицы (запомни назначение)

| Таблица | Зачем |
|---------|--------|
| `users` | Логины, пароли, ФИО, роли |
| `products` | Товары (артикул — главный ключ) |
| `pickup_points` | Адреса пунктов выдачи |
| `orders` | Заказы |
| `order_items` | Какие товары в каком заказе (если делал) |
| `roles` | Справочник ролей (опционально) |
| `order_statuses` | Справочник статусов (опционально) |

### Пошагово на экзамене (простой путь)

**Шаг 1.** pgAdmin → Create Database → `shoe_store_demo`

**Шаг 2.** Query Tool → выполнить SQL создания таблиц (из `модуль 1/Модуль 1 — финальный простой вариант.txt`)

**Шаг 3.** Импорт CSV из `tables_csv/`:
- ПКМ по таблице → Import/Export Data
- Format: csv, Header: Yes, Delimiter: `,`
- Для `pickup_points` — только колонка `full_address` (не `pickup_point_id`!)

**Шаг 4.** Заполнить `role_id`, `client_id`, `status_id` (UPDATE из инструкции модуля 1)

**Шаг 5.** Добавить FOREIGN KEY

**Шаг 6.** Проверка:
```sql
SELECT COUNT(*) FROM users;      -- около 10
SELECT COUNT(*) FROM products;   -- около 30
SELECT COUNT(*) FROM orders;     -- около 10
```

**Шаг 7.** ERD Tool → экспорт PDF

### Быстрый путь (если есть backup)

1. Query Tool → открыть `модуль 1/backup_bd.sql` → Execute
2. Проверить COUNT(*)
3. ERD → PDF

### Типичные ошибки модуля 1

| Ошибка | Решение |
|--------|---------|
| `client_id` при импорте orders | Импортировать только 8 колонок из CSV, без client_id |
| Разделитель `;` вместо `,` | Delimiter = запятая |
| pickup_points падает | Импорт только в `full_address` |
| Поля перепутаны | Не использовать SELECT * при редактировании — указывать имена колонок |

## 6. Модуль 3 — Поиск, фильтр, CRUD товаров

### Цель модуля

- Менеджер и админ: поиск, сортировка по складу, фильтр по поставщику (сразу, без кнопки «Найти»)
- Админ: добавить / изменить / удалить товар
- Одно окно редактирования
- Фото 300×200, путь в БД
- Нельзя удалить товар, если он в заказе

### Кто что видит

| Роль | Интерфейс |
|------|-----------|
| Гость, Клиент | Только список (как модуль 2) |
| Менеджер | + панель поиска, сортировки, фильтра |
| Администратор | + кнопка «Добавить товар», клик по товару |

### Ключевые функции (модуль 3)

| Функция | Что делает |
|---------|------------|
| `filter_products()` | Фильтр + поиск + сортировка в Python |
| `on_filter_change()` | Обновить список при вводе в поиск |
| `refresh_list()` | Перерисовать список товаров |
| `open_form("add")` | Окно добавления товара |
| `open_form("edit", article)` | Окно редактирования |
| `save_product()` | INSERT или UPDATE в БД |
| `delete_product()` | DELETE (если не в заказе) |
| `check_form()` | Проверка полей перед сохранением |
| `next_article()` | Новый ID товара (+1) |
| `in_orders(article)` | Есть ли товар в заказах |
| `form_open` | Флаг: только одно окно редактирования |

### Как работает поиск (простыми словами)

1. Все товары загружаются из БД: `get_products()`
2. `filter_products()` оставляет только подходящие:
   - текст из поиска есть в любом поле товара
   - поставщик совпадает (или «Все поставщики»)
   - сортировка по количеству на складе
3. `search_var.trace_add("write", ...)` — при каждой букве в поиске список обновляется

### Форма товара (поля)

| Поле | Тип |
|------|-----|
| Наименование | Entry |
| Категория | Combobox |
| Описание | Entry |
| Производитель | Combobox |
| Поставщик | Entry |
| Цена | Entry (может быть 1999.99) |
| Ед. изм. | Entry |
| На складе | Entry (целое ≥ 0) |
| Скидка % | Entry (0–100) |
| Фото | Кнопка «Выбрать фото» |

**При добавлении:** артикул не показывается, создаётся автоматически.  
**При редактировании:** артикул только для чтения.

## 8. Справочник: функция db() — сердце приложения

Все обращения к базе идут через одну функцию:

```python
def db(sql, params=None, one=False):
```

**Примеры использования:**

```python
# Одна строка (логин)
row = db("SELECT full_name, role_name FROM users WHERE login=%s AND password_plain=%s",
         (login, password), one=True)

# Много строк (товары)
rows = db("SELECT article, product_name, ... FROM products")

# Добавить
db("INSERT INTO products(...) VALUES (%s,%s,...)", (art, name, ...))

# Изменить
db("UPDATE products SET product_name=%s WHERE article=%s", (name, art))

# Удалить
db("DELETE FROM products WHERE article=%s", (article,))
```

**Важно:** в SQL используй `%s`, а значения передавай вторым аргументом — так безопасно от ошибок.

## 9. Справочник: tkinter — что где в коде

| Элемент на экране | В коде |
|-------------------|--------|
| Окно | `tk.Tk()` → переменная `win` |
| Рамка/блок | `tk.Frame(...)` |
| Текст | `tk.Label(..., text="...")` |
| Поле ввода | `tk.Entry(...)` |
| Кнопка | `tk.Button(...)` или `btn(...)` |
| Выпадающий список | `ttk.Combobox(...)` |
| Сообщение | `messagebox.showinfo / showwarning / showerror` |
| Отдельное окно | `tk.Toplevel(win)` |
| Прокрутка списка | `Canvas` + `Scrollbar` + `Frame` внутри |
| Смена экрана | `clear()` + заново создать виджеты |

**Переменная для поиска в реальном времени:**
```python
search_var = tk.StringVar()
search_var.trace_add("write", on_filter_change)
```
